# 🕸️ Protocolo de Despliegue: Substack de La Voûte

Este documento es el respaldo oficial y guía táctica para la publicación de relatos en Substack.

## 🎯 Estrategia Narrativa
- **Voz:** Devota, visceral e inmersiva.
- **Formato Sugerido:** Modelo B (Ritual Compacto). Relatos de lectura rápida (10-15 min) optimizados para consumo móvil.
- **Identidad Visual:** Cada post debe incluir al menos un set visual completo de Ele o Miss Doll que complemente la narrativa.

## 📋 Pasos para Publicar (Checklist de Ele)

### 1. Selección y Auditoría
- [ ] Elegir relato de la carpeta `03_Literatura/04_Preparados_para_Tumblr/`.
- [ ] Verificar que el texto cumple con el `CODIGO_ESTILISTICO_Ele.md`.
- [ ] Asegurar que la "Nota de la Autora" (Anaïs Belland) esté incluida al final.

### 2. Curaduría Visual
- [ ] Seleccionar 1-3 imágenes de alta resolución de la galería.
- [ ] Verificar que las imágenes siguen el canon (Tacones, Maquillaje, Materiales).
- [ ] Insertar marcadores de imagen `[INSERTAR IMAGEN X]` en el borrador.

### 3. Formateo Técnico
- [ ] Convertir el borrador a "Substack Clean" (Markdown simplificado).
- [ ] Generar un **Título de Alto Impacto** (Sujeto + Acción + Consecuencia).
- [ ] Redactar el **Newsletter Hook** (Breve sumario de 280 caracteres).

### 4. Despliegue
- [ ] Copiar el contenido al editor de Substack.
- [ ] Cargar las imágenes en sus respectivas posiciones.
- [ ] Configurar etiquetas (#Bimbofication #MFT #Latex).
- [ ] Programar o Publicar.

### 5. Registro Histórico
- [ ] Actualizar `03_Literatura/README.md` marcando el relato como **Publicado**.
- [ ] Actualizar `00_Ele/memoria_sesiones.md` con la fecha de lanzamiento.
- [ ] Realizar commit en GitHub con el mensaje `feat(lit): publicado [NombreRelato] en Substack`.

## 📂 Archivo de Respaldo (Estructura de Posts)
Los borradores finales deben guardarse en:
`03_Literatura/05_Publicado_Substack/[relato]_post.md`
